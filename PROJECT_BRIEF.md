# PROJECT BRIEF — 유도 대진표 웹앱

## 1. 프로젝트 개요

유도 경기를 중심으로 한 웹 기반 대진표 및 경기 운영 시스템을 개발한다.
초기 버전은 웹앱으로 출시하고, 추후 모바일 앱으로 확장 가능하도록 설계한다.
이 서비스는 단순 대진표 생성기가 아니라 대회 운영, 선수 등록, 경기 결과 기록, 개인 경기 이력 조회까지 포함하는 운영 플랫폼을 목표로 한다
---

## 2. 프로젝트 목표

- 유도 대회용 대진표를 자동 생성할 수 있어야 한다.
- 경기 결과를 실시간으로 입력하고 다음 라운드에 자동 반영할 수 있어야 한다.
- 경기 중 득점 상황을 어떤 기술로 얻었는지 기록할 수 있어야 한다.
- 대회 주최자, 학교/동아리 운영자, 개인 선수가 각각 적절한 화면에서 서비스를 사용할 수 있어야 한다.
- 대회별 공유 링크 및 코드 기반 접근 기능이 있어야 한다.
- 직관적인 UI와 확장 가능한 구조를 갖춰야 한다.

---

## 3. 주요 사용자

### 3.1 대회 주최자(관리자)

- 대회 생성 및 관리
- 부문/체급 구성
- 참가자 관리
- 대진 생성
- 경기 결과 입력
- 대진표 출력 및 공유

### 3.2 학교/동아리 운영자

- 소속 선수 등록 및 수정
- 대회 참가 신청 관리
- 선수 정보 유지보수

### 3.3 개인 선수

- 로그인 후 본인 참가 대회 확인
- 본인 경기 결과 및 기술 기록 조회
- 과거 경기 기록 열람

---

## 4. 화면 구조

이 프로젝트는 화면을 명확히 분리한다.

### 4.1 관리자/운영자 화면

- 대회 주최자 및 학교/동아리 운영자용
- 데이터 입력/수정/관리 중심

### 4.2 개인 선수 화면

- 선수 본인이 로그인하여 조회하는 화면
- 본인 경기 기록, 참가 대회, 결과 조회 중심

### 4.3 공개 조회 화면

- 로그인 없이 링크 또는 대회 코드로 접근 가능
- 읽기 전용 대진표 조회 화면

---

## 5. 핵심 기능

### 5.1 선수 및 소속 관리

- 선수 등록/수정/삭제
- 학교/도장/클럽 관리
- 대회 참가자 등록

### 5.2 대회 관리

- 대회 생성/수정
- 대회 코드 발급
- 공유 링크 생성
- 부문/체급/연령/성별 설정

### 5.3 대진 생성

- 자동 대진 생성
- 같은 도장/클럽 선수는 첫 경기에서 최대한 만나지 않도록 처리
- 부전승 자동 처리
- 승자 다음 라운드 자동 반영

### 5.4 경기 운영

- 경기 결과 입력
- 득점 기록 입력
- 어떤 기술로 득점했는지 기록
- 경기별 상세 이력 저장

### 5.5 조회 및 공유

- 대회 코드 입력 후 대진표 조회
- 공유 링크로 대진표 조회
- PDF 저장 및 출력
- 개인 선수의 경기 기록 조회

---

## 6. UX/UI 방향

- 관리자 화면은 복잡한 정보를 빠르게 처리할 수 있어야 한다.
- 선수 화면은 모바일에서도 쉽게 읽히는 직관적인 구조여야 한다.
- 공개 조회 화면은 빠르게 접근 가능해야 한다.
- 디자인은 깔끔하고 전문적인 스포츠 운영 툴 느낌을 지향한다.
